# 레코컬쳐 홈페이지 배포 가이드

## 전체 소요시간: 약 15분

---

## STEP 1. GitHub 계정 만들기 (이미 있으면 스킵)

1. https://github.com 접속
2. **Sign up** 클릭
3. 이메일, 비밀번호, 사용자명 입력 후 가입

---

## STEP 2. 새 Repository 만들기

1. GitHub 로그인 후 우측 상단 **+** 버튼 → **New repository**
2. 아래와 같이 설정:
   - **Repository name**: `recoculture-site`
   - **Public** 선택 (GitHub Pages 무료 사용 조건)
   - ✅ **Add a README file** 체크
3. **Create repository** 클릭

---

## STEP 3. 파일 업로드

1. 생성된 repository 페이지에서 **Add file** → **Upload files** 클릭
2. 다운로드 받은 파일 3개를 드래그앤드롭:
   - `index.html` (메인 홈페이지)
   - `CNAME` (도메인 연결용)
   - `404.html` (에러 페이지)
3. 하단 **Commit changes** 클릭

---

## STEP 4. GitHub Pages 활성화

1. Repository 상단 **Settings** 탭 클릭
2. 좌측 메뉴에서 **Pages** 클릭
3. **Source** 섹션에서:
   - Branch: **main** 선택
   - Folder: **/ (root)** 선택
4. **Save** 클릭
5. 잠시 후 상단에 초록색으로 사이트 URL이 표시됨

---

## STEP 5. 호스팅케이알 DNS 설정

> ⚠️ 이 단계가 가장 중요합니다. recoculture.com이 GitHub Pages를 바라보도록 설정합니다.

1. https://hosting.kr 로그인
2. **도메인 관리** → **recoculture.com** 선택
3. **DNS 관리** 또는 **네임서버/DNS 설정** 메뉴 진입
4. **기존 A 레코드, CNAME 레코드가 있으면 삭제** (아임웹 연결 해제)
5. 아래 레코드를 **새로 추가**:

### A 레코드 (4개 모두 추가)

| 타입 | 호스트 | 값 |
|------|--------|-----|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |

### CNAME 레코드 (1개)

| 타입 | 호스트 | 값 |
|------|--------|-----|
| CNAME | www | {GitHub사용자명}.github.io |

> `{GitHub사용자명}` 부분을 본인의 GitHub 사용자명으로 교체하세요.
> 예: GitHub 사용자명이 `recoculture`이면 → `recoculture.github.io`

6. **저장** 클릭

---

## STEP 6. HTTPS 활성화

> DNS 반영까지 최대 24시간 걸릴 수 있습니다 (보통 10분~1시간)

1. GitHub repository → **Settings** → **Pages**
2. **Custom domain** 란에 `recoculture.com` 입력 → **Save**
3. DNS 확인이 완료되면 **Enforce HTTPS** 체크박스가 나타남 → ✅ 체크

---

## STEP 7. 확인

브라우저에서 아래 주소로 접속해서 확인:
- https://recoculture.com
- https://www.recoculture.com

---

## 이후 수정이 필요할 때

1. GitHub repository에서 `index.html` 클릭
2. 연필 아이콘 (✏️ Edit) 클릭
3. 수정 후 **Commit changes**
4. 1~2분 후 자동 반영

또는 저한테 수정 요청하시면 수정된 `index.html`을 다시 드릴게요.

---

## 문제 해결

| 증상 | 해결 |
|------|------|
| 사이트 접속 안 됨 | DNS 반영 대기 (최대 24시간) |
| "404 There isn't a GitHub Pages site" | Settings → Pages에서 branch가 main으로 설정되어 있는지 확인 |
| HTTPS 안 됨 | DNS 완전 반영 후 Enforce HTTPS 체크 |
| www 접속 안 됨 | CNAME 레코드 확인 |

---

## 비용

- GitHub Pages: **무료**
- 도메인 유지비: 기존 호스팅케이알 비용만 (연 ~1~2만원)
- SSL(HTTPS): **무료** (GitHub 제공)
- 아임웹: **해지 가능** → 월 비용 절감
